Architecture Decision: Keep Squarespace or Switch?

Understand what you're working with before touching anything.

5 min read

The short answer: keep your Squarespace domain, host files on Vercel

You own the domain stragentech.com through Squarespace. That's a DNS registration — it's separate from website hosting. Think of it like owning a phone number (Squarespace) vs. where calls get routed (Vercel).

The two-part setup you'll build:

  • stragentech.com (main site) — hosted on Vercel, your Squarespace domain points to it
  • apps.stragentech.com (apps hub) — also hosted on Vercel, a subdomain you create in Squarespace DNS

Squarespace itself won't host your custom HTML — it only serves Squarespace-built pages. So the domain stays there, the files live on Vercel.

Why Vercel specifically?

PlatformFree TierCustom DomainSubdomainsAPI/Backend FunctionsBest For
Vercel ✓ recommended ✓ generous ✓ yes ✓ yes ✓ built-in Edge Functions Your use case — static site + API proxy
Netlify ✓ generous ✓ yes ✓ yes ✓ serverless functions Also good, slightly more complex proxy setup
Squarespace hosting ✗ ✓ built-in ✗ no custom HTML subdomains ✗ no backend Squarespace-only sites, not custom HTML
GitHub Pages ✓ ✓ yes ✓ yes ✗ static only, no proxy Static sites only, no API key needed

Vercel wins because you need the API proxy function to keep your Anthropic key private. GitHub Pages can't do that.

What you'll need before starting

Your Anthropic API key — get one at console.anthropic.com (free to create, pay per use)

GitHub Setup

Create a repository and upload your project files. GitHub is the source of truth — Vercel watches it and auto-deploys when you make changes.

~10 min
What is GitHub? Think of it as Dropbox for code files, with version history. When you push a file change to GitHub, Vercel automatically picks it up and republishes your site within 30 seconds. You never need to manually redeploy.
Step 1.1

Create a GitHub account (if you don't have one)

  1. Go to github.com and click Sign up
  2. Enter your email, create a password, choose a username (e.g. bilalkhan-tech)
  3. Verify your email — GitHub sends a code
  4. On the welcome screen, choose Free plan
Step 1.2

Create a new repository

  1. Click the + icon (top right) → New repository
  2. Fill in the form:
    • Repository name: stragentech-site
    • Description: Stragentech.com website and apps hub
    • Visibility: Private (recommended — keeps your code private)
    • Initialize with README: ✓ check this box
  3. Click Create repository
You now have an empty repository at github.com/YOUR-USERNAME/stragentech-site
Step 1.3

Upload your files to GitHub

Your repository needs exactly this file structure:

stragentech-site/ ├── index.html ← rename of stragentech.html (main site) ├── apps.html ← the apps hub (proxy-ready version) ├── vercel.json ← routing configuration ├── api/ │ └── analyze.js ← the API proxy function └── README.md ← already there from initialization

How to upload via GitHub's web interface (no coding required):

  1. In your new repository, click Add file → Upload files
  2. Drag and drop all your files at once. For the api/ folder: GitHub's web uploader doesn't create folders directly — use this workaround:
Folder workaround: When uploading analyze.js, after clicking Upload Files, type api/analyze.js as the filename in the "name" field at the top of the file — GitHub will automatically create the api/ folder. Alternatively, use the method below.

Alternative: Upload analyze.js directly via GitHub's file editor

  1. In your repository, click Add file → Create new file
  2. In the filename box, type: api/analyze.js — GitHub creates the folder automatically when you type the slash
  3. Paste the entire contents of analyze.js into the editor
  4. Scroll down, click Commit new file
  5. Repeat for vercel.json, index.html, and apps.html
Important rename: The main site file must be named index.html (not stragentech.html). Vercel serves index.html as the default page at the root URL. Rename it before uploading.
Step 1.4

Verify your repository structure

After all uploads, your GitHub repository should look like this at github.com/YOUR-USERNAME/stragentech-site:

📁 api/ · 📄 apps.html · 📄 index.html · 📄 vercel.json · 📄 README.md

Click into api/ and confirm analyze.js is there.

Vercel Deployment

Connect Vercel to your GitHub repo, set your API key, and get a live URL.

~15 min
What is Vercel? It's a hosting platform that watches your GitHub repository and automatically publishes your site every time you make a change. It also runs your api/analyze.js function as a serverless backend — this is what keeps your API key private.
Step 2.1

Create a Vercel account

  1. Go to vercel.com
  2. Click Sign Up
  3. Choose Continue with GitHub — this links the two accounts automatically. Authorize when prompted.
  4. On the "Tell us about yourself" screen: select Personal → Hobby (free)
Step 2.2

Import your GitHub repository into Vercel

  1. From your Vercel dashboard, click Add New… → Project
  2. You'll see a list of your GitHub repositories. Find stragentech-site and click Import
  3. On the configuration screen:
    • Project Name: stragentech-apps (this becomes your temporary URL: stragentech-apps.vercel.app)
    • Framework Preset: leave as Other
    • Root Directory: leave blank (uses root)
    • Build & Output Settings: leave all blank — your site is plain HTML, no build needed
  4. Do NOT click Deploy yet — first set the environment variable in the next step
If you don't see your repo: Click "Adjust GitHub App Permissions" and grant Vercel access to the stragentech-site repository specifically.
Step 2.3

Add your Anthropic API key as an environment variable

This is the most important security step. Your API key goes into Vercel's encrypted environment — never into your code files.

Get your Anthropic API key first:

  1. Go to console.anthropic.com → sign up or log in
  2. Click your account name (top right) → API Keys
  3. Click Create Key, give it a name like stragentech-production
  4. Copy the key — it starts with sk-ant-api03-...
  5. Save it somewhere safe immediately — Anthropic only shows it once

Add the key to Vercel:

  1. Back on the Vercel import screen, scroll down to Environment Variables
  2. Click Add and enter:
    • Name: ANTHROPIC_API_KEY (exact — uppercase, underscores)
    • Value: paste your API key
    • Environments: check Production, Preview, and Development
  3. Click Add to save it
Never put your API key directly in HTML or JavaScript files. If it's in your code, anyone who views your page source can steal it and run up charges on your account. The environment variable method keeps it server-side only.

Now click Deploy. Vercel will build and deploy in about 30–60 seconds.

Step 2.4

Verify the deployment works

  1. After deploy completes, Vercel shows a success screen with a confetti animation and a preview URL like stragentech-apps.vercel.app
  2. Click Visit — your main site should load with the Stragentech design
  3. Add /apps.html to the URL to visit the apps hub
  4. Test a live tool: Go to the Workflow Analyzer, paste a short description of a process, and click Analyze. You should see the AI response appear within a few seconds.
✓ If a tool returns an AI response: Your proxy is working. The API key is secure. You have a live, functional site.
If a tool returns an error: The most common cause is a typo in the environment variable name. Go to Vercel Dashboard → Your Project → Settings → Environment Variables → confirm ANTHROPIC_API_KEY is spelled exactly right, then go to Deployments → click the three dots on the latest deployment → Redeploy.

DNS: Connect Your Squarespace Domain

Point stragentech.com and apps.stragentech.com at your Vercel deployment.

~20 min setup + 24–48h propagation
What is DNS? DNS (Domain Name System) is the phone book of the internet. When someone types stragentech.com, DNS looks up what server to connect them to. You're about to update that lookup to point to Vercel.
Step 3.1

Add your domain in Vercel first

  1. In your Vercel project dashboard, click Settings → Domains
  2. In the "Add Domain" field, type stragentech.com and click Add
  3. Vercel will show you a warning: "Domain is not configured." That's expected — you haven't updated DNS yet.
  4. Vercel shows you the DNS record you need to add. It will look like one of these: 
    Type: A
Name: @
Value: 76.76.21.21

— OR —

Type: CNAME
Name: www
Value: cname.vercel-dns.com
  5. Copy these values — you'll paste them into Squarespace in the next step
  6. Also click Add for www.stragentech.com — Vercel will show a CNAME record for this too
Step 3.2

Update DNS in Squarespace

  1. Log into your Squarespace account at squarespace.com
  2. Click your account icon (top right) → Domains
  3. Find stragentech.com and click on it
  4. Click DNS Settings (or "DNS" in the left sidebar)
  5. You'll see existing DNS records. You need to edit or add the following: 
    Record 1 — Root domain:
  Type:  A
  Host:  @
  Value: 76.76.21.21    ← use the value Vercel gave you

Record 2 — www subdomain:
  Type:  CNAME
  Host:  www
  Value: cname.vercel-dns.com    ← use the value Vercel gave you
  6. If a conflicting record already exists (Squarespace often adds their own A record), delete or overwrite it with the Vercel value
  7. Click Save
Squarespace note: Squarespace DNS UI varies slightly by account type. Look for a section labeled "Custom Records" or "DNS Settings." If you only see Squarespace's own records and can't edit them, go to Account → Billing → check if DNS management is enabled. It should be on by default for domains registered through Squarespace.
Step 3.3

Create the apps.stragentech.com subdomain

Still inside Squarespace DNS Settings, add one more record:

Record 3 — Apps subdomain:
  Type:  CNAME
  Host:  apps
  Value: cname.vercel-dns.com

Then in Vercel → Settings → Domains, add apps.stragentech.com the same way you added the root domain.

How this routes traffic: When someone visits apps.stragentech.com, Squarespace DNS says "go to Vercel." Vercel checks its domain list, sees you've registered apps.stragentech.com, and serves your apps.html file. But wait — Vercel will serve index.html by default, not apps.html. See the routing fix below.
Step 3.4

Fix routing so apps.stragentech.com serves apps.html

Update your vercel.json file in GitHub to route the subdomain correctly:

{
  "version": 2,
  "name": "stragentech-apps",
  "routes": [
    {
      "src": "/api/analyze",
      "dest": "/api/analyze.js"
    },
    {
      "src": "/(.*)",
      "dest": "/$1"
    }
  ]
}

Then update your apps.html file: the navigation link href="stragentech.html" should become href="/" or href="https://stragentech.com", and index.html should use href="https://apps.stragentech.com/apps.html" for the Apps nav link.

Simpler approach: Vercel also lets you set a custom "root directory" per domain. In Settings → Domains, after adding apps.stragentech.com, Vercel will ask which page to serve. Point it to /apps.html.
Step 3.5

Wait for DNS propagation and verify

DNS changes take anywhere from 5 minutes to 48 hours to propagate worldwide. Typically it's under 1 hour.

To check propagation status:

  1. Go to dnschecker.org
  2. Enter stragentech.com and select A record type
  3. When most or all checkmarks show green with the Vercel IP, your domain is live

Meanwhile, Vercel automatically provisions a free SSL certificate (the padlock / https://). This takes a few minutes after DNS propagates.

✓ Success state: You type https://stragentech.com in a browser and your site loads with a padlock. You type https://apps.stragentech.com/apps.html and the apps hub loads. A live tool runs and returns an AI response.

Ongoing: Making Updates (2 min per change)

How to update your site after it's live — no developer needed.

2 min per update

The update workflow: GitHub → auto-deploys to Vercel

  1. Go to github.com/YOUR-USERNAME/stragentech-site
  2. Click on the file you want to edit (e.g. index.html)
  3. Click the pencil icon (Edit this file) in the top right of the file view
  4. Make your changes — update text, add a section, change a price, etc.
  5. Scroll down to Commit changes — add a brief note like "Update services pricing"
  6. Click Commit changes
  7. Within 30 seconds, Vercel detects the change and redeploys automatically
  8. Refresh your live site — the change is live
That's the full workflow. No FTP, no build process, no command line. Edit in browser → commit → live in 30 seconds.

Adding a new blog post or insight article

The current site has placeholder blog cards. When you're ready to publish real articles:

  1. The easiest path: create a new HTML file (e.g. insights/fraud-prevention.html) using the same design system
  2. Upload to GitHub → Vercel deploys it automatically
  3. Update the blog card in index.html to link to it and remove the "Coming soon" label

Alternatively, link your LinkedIn articles or a Substack publication from the blog cards — that requires only changing a few URLs in the HTML.

Rotating your Anthropic API key

If your API key is ever compromised (or you just want to rotate it periodically):

  1. Generate a new key at console.anthropic.com → API Keys → Create Key
  2. In Vercel: Project → Settings → Environment Variables → find ANTHROPIC_API_KEY → click Edit → paste new value → Save
  3. Go to Deployments → Redeploy the latest deployment (to apply the new key)
  4. Delete the old key in Anthropic console

Troubleshooting Common Issues

Most problems have simple fixes.

Live tool returns an error or blank output

  • Most likely: ANTHROPIC_API_KEY environment variable is missing or misspelled. Check Vercel → Settings → Environment Variables.
  • After fixing an env var, you must redeploy: Vercel → Deployments → three dots → Redeploy
  • Check Vercel Function Logs: Project → Functions tab → click on api/analyze → view error messages

Domain shows Vercel's default page or "404"

  • DNS hasn't propagated yet — wait up to 48 hours and check again at dnschecker.org
  • The domain wasn't added in Vercel's domain settings — go to Project → Settings → Domains and add it
  • The file is named incorrectly — the root URL serves index.html only. Make sure your main site file is named exactly index.html in GitHub.

"Not Secure" warning / no padlock

  • Vercel auto-provisions SSL but it takes a few minutes after DNS propagates
  • If it's been more than 1 hour: Vercel → Project → Settings → Domains → click "Refresh" on the domain
  • The www. redirect: add both stragentech.com and www.stragentech.com in Vercel domains to ensure both work with https

GitHub upload shows error: "file too large"

  • GitHub has a 25MB file size limit for web uploads and 100MB for the repo. Your HTML files are well under 1MB, so this shouldn't occur.
  • If you ever add large assets (images, PDFs): host them on a CDN like Cloudflare Images or store in a GitHub Releases attachment instead of the repo itself.

Squarespace DNS — can't find DNS settings

  • Navigation varies: try Domains → click your domain → DNS Settings, OR Settings → Domains → [your domain] → DNS panel
  • If Squarespace shows "Domain managed by third party" — your domain may be using external nameservers. In that case, add the DNS records wherever your nameservers are managed.
  • Squarespace support (support.squarespace.com) has live chat that can walk you to the DNS panel.
Still stuck? Vercel's documentation at vercel.com/docs is excellent and beginner-friendly. Their support chat (free tier included) typically responds within a few hours.
